# Build In Public Plan

POSA will be public from day one.

## Goals

- Build trust through visible progress.
- Attract contributors and testers.
- Show Unplugged AI can ship offline-first field software.
- Keep expectations honest while the app is early.

## Public Channels

Primary:

- GitHub repository.
- GitHub Issues.
- GitHub Discussions when useful.
- Release notes.

Optional:

- X/Twitter updates.
- Reddit posts after the first runnable prototype.
- Short demo videos.
- Blog/devlog posts.

## Community Metrics

Use:

- Stars.
- Forks.
- Issues.
- Discussions.
- Pull requests.
- Release downloads.
- Direct feedback.

Do not use hidden analytics. Optional crash reports or feedback bundles may be considered later only with explicit opt-in.

## Public Messaging

Recommended short description:

> POSA is a free, open-source offline survival companion for Android, built by Unplugged AI. It combines offline maps, field tools, notes, checklists, local guide packs, and source-grounded guidance without requiring cloud services or cell signal.

Always keep public messaging clear that POSA is early, offline-first, and safety-conscious.
