# Contributing

POSA is public from day one, but the project is intentionally scoped. Contributions should strengthen the offline-first Android app, source-grounded content system, maps foundation, field tools, notes, or documentation.

## Principles

- Keep POSA useful without internet, accounts, cloud services, or AI.
- Prefer small, reviewable pull requests.
- Keep safety and provenance visible.
- Do not add hidden telemetry.
- Do not add medical treatment content without an explicit source and review plan.
- Do not add government/customer-specific workflows to the public app.

## Good First Areas

- Documentation improvements.
- Pack schema examples.
- UI polish for existing screens.
- Local storage tests.
- Offline map import test cases.
- Accessibility and field-readability improvements.

## Before Opening A Pull Request

- Explain the user-facing behavior.
- Note any privacy, safety, map licensing, or content provenance impact.
- Include tests when changing app behavior.
- Keep generated files and unrelated formatting churn out of the PR.

## Content Contributions

Content contributions must include:

- Source name.
- Source URL or publication reference.
- License.
- Pack ID and version.
- Review status.
- Last reviewed date.

Unreviewed content must be labeled as unreviewed. User-imported and community packs must never appear equivalent to official reviewed packs.
